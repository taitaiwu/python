# 作業四 : 加權中位數與一維站點選址（Prune-and-Search）

**一、目標與學習重點**
本題要求以 Prune-and-Search（剪枝搜尋）方法，在未進行整體排序的前提下，求解加權中位數（Weighted Median），並應用於一維 L1 站點選址問題。透過此題，學生需：
* 理解加權中位數的定義與性質；
* 設計能在每一輪「剪除常數比例元素」的演算法；
* 以程式實作並在大型輸入下驗證線性時間等級的成長趨勢。

**二、問題敘述**
給定 n 個點 ($x_i, w_i$)（其中 $w_i > 0$），定義加權中位數 $m$ 滿足：左側權重總和 ≤ 0.5，右側權重總和 ≤ 0.5（總權重可視需要正規化為 1）。
已知一維 L1 站點選址問題 $\min_x \sum_{\forall i} w_i|x-x_i|$ 的最佳解即為加權中位數。請以 Prune-and-Search 方法求得 m，並輸出 m 與對應之目標值 $\min_x \sum_{\forall i} w_i|x-x_i|$。

**三、輸入與輸出規格（標準輸入／輸出）**
**輸入（stdin）：**
1. 第一行：整數 $n$
2. 接續 $n$ 行：每行兩個數 $x_i$, $w_i$（$x_i$, $w_i$ 可為浮點數，且 $w_i > 0$）

**輸出（stdout）：**
1. 第一行：加權中位數 $m$（可為浮點數；若存在加權中位數區間，可輸出其中任一值）
2. 第二行：目標值 $\min_x \sum_{\forall i} w_i|x-x_i|$ （浮點數，請控制合適的小數位或採用合理的格式化）

**四、範例**
**輸入：**
```
5
0 1
10 2
20 3
21 1
30 3
```
**輸出（其中一種可能）：**
```
20
```
（請由程式計算加權 L1 總和；示意值，依資料計算為準）

**五、限制與要求**
1. 不得整體排序：不可使用任何會導致整體排序的函式或容器操作（例如 Python 的 `sorted`、`list.sort`；C++ 的 `std::sort` 等）。
2. 必須採用 Prune-and-Search：演算法需在每一輪剪除常數比例元素（請在程式註解或 REPORT.md 中簡述不變量與剪枝比例）。
3. 時間複雜度：理想為 $O(n$) 期望或保證時間。
4. 數值處理：$x_i$ 與 $w_i$ 可為浮點數；請注意累加誤差與輸出格式。

**六、實作提示（非強制）**
* 可類比「Median of Medians」的分組取代表法，為每輪比較與剪枝提供良好 pivot。
* 可先將總權重正規化為 1，並維護左右權重和，據以決定保留或剪除的範圍。
* 當多個 $x_i$ 相同時，需合併其權重以正確計算裁決條件（≤ 0.5）。
* 允許在小規模子陣列（例如長度 ≤ 5）中使用常數時間排序作為子步驟。

**七、繳交內容**
1. 程式碼：`main.py` 或對應語言單一檔，從 stdin 讀取、輸出至 stdout。

**八、學術誠信與限制**
* 禁用排序 API：將進行自動掃描（例如 `sorted`, `.sort`, `std::sort` 等）。
* 可能進行兩兩程式碼相似度比對（n-gram Jaccard）。
* 嚴禁抄襲或未註明引用之程式碼，違者依 0 分。
