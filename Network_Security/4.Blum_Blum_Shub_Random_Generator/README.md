# 作業四：Blum Blum Shub (BBS) 亂數產生器實作

## 題目說明

請實作 Blum Blum Shub, BBS 偽亂數位元產生器，產生長度為 10000 bits 的二進位亂數序列，並使用常見的亂數測試檢查其隨機性。

實作步驟如下：
1. 實作 BBS 亂數產生器。
2. 產生 10000-bit binary sequence。
3. 實作並通過多個常見 randomness tests。
4. 分析 3 組參數與 seed 對測試結果的影響

## 建議測試參數

### 輸入

可自行選擇 `p`、`q`、`seed`，但必須符合 BBS 條件。也可以使用以下參考參數：
```
p=3, q=7, seed=4, n=10000
```
```
p = 1000003, q = 2000003, seed = 271828182, n = 10000
```

### 輸出

請將 10000 bits 輸出為文字檔： `bbs_output.txt` 檔案中應只包含 0 與 1，不可包含空白、逗號或其他符號

## 測試內容

### Test 1：0 和 1 的數量是否接近
請計算：
```
number_of_0
number_of_1
```
* 因為總長度是 10000 bits，所以理想情況大約是：0 約 5000 個； 1 約 5000 個
* 通過標準： 4800 ≤ `number_of_0 `≤ 5200 and 4800 ≤ `number_of_1` ≤ 5200

### Test 2：連續相同 bit 的長度是否太長
例如：`000000000000000` 代表出現很長一段連續 `0`。請找出最長連續相同 bit 的長度： `max_run_length`
* 通過標準：`max_run_length` ≤ 20
* 說明： 10000 bits 的亂數中，偶爾出現 10 幾個連續相同 bit 是正常的，但如果出現 50 個以上，通常就很可疑。

### Test 3：每 100 bits 是否大致平衡
將 10000 bits 分成 100 組，每組 100 bits。例如：
```
第 1 組：第 1 ~ 100 bits
第 2 組：第 101 ~ 200 bits
...
第 100 組：第 9901 ~ 10000 bits
```
* 對每一組計算 `1` 的數量。理想情況每組大約有 50 個 `1`。
* 通過標準：每一組的 `1` 的數量大多落在 35 到 65 之間
* 建議判定：至少 95 組通過(也就是 100 組中至少 95 組的 `1` 的數量介於 35 到 65)

### Test4：`00`、`01`、`10`、`11` 出現次數是否接近
將 bit sequence 每 2 bits 分成一組。 例如：`10110001...` 分成 `10` `11` `00` `01` ...
* 10000 bits 可以分成 5000 組。請計算以下四種 pattern 的出現次數：`00` `01` `10` `11`
* 理想情況每一種大約出現：5000 / 4 = 1250 次
* 通過標準：1000 ≤ 每一種 pattern 的出現次數 ≤ 1500

### Test 5：前半段與後半段是否太像
將 10000 bits 分成兩半：
```
前 5000 bits
後 5000 bits
```
比較兩段在相同位置上是否相同。例如：
```
前半段：10110
後半段：10011
```
逐位比較：
```
1 = 1   → 相同
0 = 0   → 相同
1 != 0  → 不同
1 = 1   → 相同
0 != 1  → 不同
```
* 請計算相同的數量：`same_count`
* 理想情況大約一半相同，也就是 `same_count` 約 2500
* 通過標準： 2300 ≤ `same_count` ≤ 2700

## 報告要求

學生報告只需要包含：
1. 使用的 `p`, `q`, `seed`, `N`
2. 前 100 bits
3. `0` 的數量與 `1` 的數量
4. 最長連續相同 bit 長度
5. 每 100 bits 分組後，有幾組通過平衡測試
6. `00`, `01`, `10`, `11` 的出現次數
7. 前半段與後半段相同位置的數量
8. 哪些測試通過，哪些測試失敗
